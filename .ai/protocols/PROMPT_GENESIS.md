| المحور | الحد الأدنى للقبول |
|--------|-------------------|

**Agent D · Security Engineer · [Mode: Security Audit]:**
* افحص الكود القائم:
  - SQL injection, exposed secrets, auth gaps.
  - بيانات حساسة غير محمية.
* أصدر "تقرير الأمان الأولي".

**Agent E · الناقد · [Mode: Legacy Critique]:**
* انتقد الكود القائم:
  - الديون التقنية الحرجة.
  - الكود الميت (dead code).
  - الأنماط الخاطئة الشائعة.
* ركّز على المشاكل الحقيقية — لا تبالغ.

[Phase 3 — القرار (Decision)]

**Agent F · محكّم النقد · [Mode: Keep / Discard / Rewrite]:**
لكل مكون في الكود القائم:

| الحكم | المعنى | المعيار |
|-------|--------|---------|
| ✅ KEEP | احتفظ به كما هو | يعمل + آمن + موثق |
| 🔄 REFACTOR | احتفظ بالمنطق، أعد الكتابة | منطق صحيح، تطبيق ضعيف |
| ⚠️ REWRITE | أعد كتابته من الصفر | منطق مشكوك فيه أو غير قابل للصيانة |
| ❌ DISCARD | احذفه | لا قيمة أو يسبب مشاكل |

**Agent 0 · المايسترو · [Mode: Strategic Decision]:**
* راجع أحكام Agent F.
* أصدر "خطة التأسيس":
  - ما يُحتفظ به فوراً.
  - ما يُعاد كتابته (وبأي ترتيب).
  - ما يُحذف.
  - أولى المهام بعد التأسيس.

[Phase 5 — التسليم (Handoff to Normal Protocols)]

**Agent 0 · المايسترو (يُغلق بروتوكول Genesis):**
* أعلن انتهاء بروتوكول Genesis.
* أبلغ المستخدم:
  - ملخص ما تم: ما بُني / ما احتُفظ به / ما حُذف.
  - الملفات التي أُنشئت.
  - أولى المهام المقترحة مرتبة حسب الأولوية.
  - "الجلسة القادمة تبدأ بـ PROMPT_INIT."

**من هذه اللحظة:**
```
كل رسالة → .cursorrules يستدعي .ai/AGENT_VACCINE.md تلقائياً
كل جلسة  → PROMPT_INIT (عبر اللقاح أو يدوياً)
كل تعديل → PROMPT_SURGICAL
كل نهاية → PROMPT_SESSION_CLOSE (اختياري — اللقاح يكفي لغالبية الحالات)
```
